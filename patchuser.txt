### Jules's Regression Fix - 2025-07-30

#### Problem
My previous change introduced a critical bug that caused the application to crash on startup in development mode. The error was `ENOENT: no such file or directory, open '...\\undefineddev-app-update.yml'`.

The crash was caused by my enhancement to the logging system. I added code at the very top of `src/main.js` that used `process.cwd()` to create a log directory. This call was executed before the Electron `app` was fully ready. In some environments, calling `process.cwd()` this early can be unreliable and was returning a path that ended with the string "undefined". This corrupted path was then used by the auto-updater later during initialization, causing the crash.

#### Solution: Path Hardening
To fix this, I performed a full audit of `src/main.js` and replaced *all* usages of `process.cwd()` with more robust, context-appropriate alternatives:
1.  **Auto-Updater Path:** Replaced `process.cwd()` with `app.getAppPath()`. This code runs inside `app.on('ready', ...)` where `app` is guaranteed to be available.
2.  **File/Directory Paths:** Replaced `process.cwd()` in all IPC handlers (`navigate`, `get-file-content`) and helper functions (`getContents`) with `app.getAppPath()`.
3.  **Development Logging:** Moved the initialization of the development log stream from the top of the file to inside the `app.on('ready', ...)` block. This ensures it also uses the safe `app.getAppPath()` and only runs when the application is fully initialized.

These changes ensure that all file paths are constructed reliably, fixing the crash and making the application more stable overall.
